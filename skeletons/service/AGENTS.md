# AGENTS.md — сервис

Репозиторий одного независимо поставляемого сервиса. Общий цикл —
`<methodology-repo>/docs/WORKFLOW.md`, архитектурные правила —
`<methodology-repo>/docs/ARCHITECTURE.md`.

## Правила

- Выполняй только текущую задачу из `<hub>/BACKLOG.md`.
- Работай в `feat/TASK-NNNN-<slug>` через PR; прямой коммит в `main` запрещён.
- Один сервис использует только один из Python, Go или Rust.
- Не меняй `service_name`, `service_language` или `service_modules` в
  `.methodology.yml` иначе как вместе с явно заданным архитектурным изменением.
- Каждый модуль следует обязательным границам `domain`, `application`, `ports`,
  `adapters` из архитектурного канона; зависимости направлены внутрь.
- Не создавай фиктивные интерфейсы, реализации или TODO ради заполнения структуры.
- Не создавай репозитории и не меняй назначение или высокоуровневую границу
  сервиса без явного решения человека.
- Не выполняй прямые межсервисные вызовы: взаимодействие сервисов идёт
  только через брокер.
- Обновляй код, тесты, `docs/ARCHITECTURE.md` и обязательную краткую спеку каждого
  затронутого самостоятельного модуля вместе.
- Публичные API и события имеют версионируемые схемы и контрактные тесты.
- Секреты, `.env` и файлы блокировки зависимостей не меняй без разрешения.
- ADR хранится только в `<hub>/adr/` и нужен для труднообратимого системного
  решения.
- Временная заготовка или TODO явно помечается и не считается готовым результатом.
- Все диаграммы оформляются только блоками `mermaid`; текстовые псевдосхемы и
  другие языки диаграмм запрещены.

## Команды

Оставь одну строку выбранного стека и удали остальные.

| Стек | Проверка | Тест | Сборка |
|---|---|---|---|
| Python | `ruff format --check . && ruff check . && pyright` | `pytest` | `uv build` |
| Go | `gofmt -l . && go vet ./...` | `go test ./...` | `go build ./...` |
| Rust | `cargo fmt --check && cargo clippy -- -D warnings` | `cargo test` | `cargo build --release` |

Перед слиянием обязательны средство проверки методологии, команды выбранного стека и
применимые проверки из `<methodology-repo>/docs/REFERENCE.md`.

## Git

Формат Conventional Commits с областью модуля или `deploy`/`docs`; слияние со свёрткой коммитов после
успешной обязательной проверки. Развёртывание в тестовой среде идёт по SHA коммита. Тег выпуска создаётся только
отдельной задачей.
