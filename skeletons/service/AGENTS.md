# AGENTS.md — сервис

Репозиторий одного независимо поставляемого сервиса. Общие полномочия и запреты,
рабочий цикл, архитектурный канон и правила поставки — в закреплённой
методологии: `<methodology-repo>/AGENTS.md` и `<methodology-repo>/docs/`.
Ниже только локальные правила сервиса.

## Локальные правила

- Один сервис использует только один из Python, Go или Rust.
- Не меняй `service_name`, `service_language` или `service_modules` в
  `.methodology.yml` иначе как вместе с явно заданным архитектурным изменением.
- Каждый модуль следует обязательным границам `domain`, `application`, `ports`,
  `adapters`; зависимости направлены внутрь; межмодульный вызов — только через
  входной порт целевого модуля.
- Межсервисное взаимодействие — только через брокер; прямые межсервисные вызовы
  запрещены.
- Публичные API и события имеют версионируемые схемы и контрактные тесты.
- Обновляй код, тесты, `docs/ARCHITECTURE.md` и краткую спеку каждого
  затронутого самостоятельного модуля вместе.

## Команды

Оставь одну строку выбранного стека и удали остальные.

| Стек | Проверка | Тест | Сборка |
|---|---|---|---|
| Python | `ruff format --check . && ruff check . && pyright` | `pytest` | `uv build` |
| Go | `gofmt -l . && go vet ./...` | `go test ./...` | `go build ./...` |
| Rust | `cargo fmt --check && cargo clippy -- -D warnings` | `cargo test` | `cargo build --release` |
