# ADR 0001 — Записывать архитектурные решения

Date: <YYYY-MM-DD>
Status: Accepted

## Контекст

Нужен способ фиксировать значимые архитектурные решения интерфейса и причину
их принятия. Без этого непонятно, почему интерфейс устроен так, и есть риск
откатить решение, не зная обоснования.

## Решение

Завести каталог ADR (`docs/adr/`). Каждое решение — отдельный файл с
однострочной нумерацией (`000N-<slug>.md`). Структура: Контекст / Решение /
Последствия. Старые решения помечаем `Superseded`, не удаляем — история должна
оставаться читаемой.

## Последствия

- Появляется трейл решений: видно, что менялось и почему.
- Расхождение доков или спорное изменение → повод для нового ADR.
- Затраты: один файл на значимое решение. Окупается читаемостью для агентов.
