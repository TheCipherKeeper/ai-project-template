# <interface>

Репозиторий клиентского приложения. Его страницы, компонентная структура,
доверительная граница и потребляемые маршруты сервиса-шлюза канонически описываются в
`docs/ARCHITECTURE.md`.

## Первичная настройка

Выполните один раз в указанном порядке:

1. Получите у человека назначение интерфейса и целевой сервис-шлюз из хаба.
2. В `.methodology.yml` закрепите точный тег или коммит методологии.
3. Заполните `docs/ARCHITECTURE.md`, включая полный манифест потребляемых
   маршрутов API; не повторяйте этот реестр в README.
4. В `.env.example` задайте только публичные имена клиентской конфигурации.
5. Зафиксируйте в `AGENTS.md` фактические команды форматирования, статического
   анализа, проверки типов, тестирования и сборки.
6. Замените маркер поставки в `.pipeline.json`; `deploy` должен завершаться
   проверкой готовности и быстрой проверкой.
7. Задайте переменные CI `METHODOLOGY_REPOSITORY`, `METHODOLOGY_VERSION`,
   `AGENT_REVIEWER_LOGIN` и `HUMAN_REVIEWER_LOGIN`; настройте среды `test` и
   защищённую `critical-test`.
8. Укажите лицензию ниже и выполните локальные проверки.

## Карта файлов

| Файл | Назначение | Режим изменения |
|---|---|---|
| `docs/ARCHITECTURE.md` | единственный реестр границ интерфейса, страниц и маршрутов сервиса-шлюза | вместе с интерфейсом |
| `.methodology.yml` | тип репозитория и закреплённая методология | при создании или обновлении методологии |
| `AGENTS.md` | ограничения клиента и точные команды | вместе с набором инструментов |
| `.env.example` | имена публичной клиентской конфигурации | вместе с конфигурационным контрактом |
| `.pipeline.json`, `.github/workflows/verify.yml` | команды и исполнимый продуктовый конвейер | вместе с инструментами и поставкой |

Системный состав и задачи находятся в хабе. Интерфейс обращается только к
зарегистрированному клиентскому сервису-шлюзу и не подключается к брокеру.

## Локальная разработка и сборка

```bash
pnpm dev
pnpm lint && tsc --noEmit
pnpm test
pnpm build
```

Рабочий цикл — `<methodology-repo>/docs/WORKFLOW.md`, поставка статики —
`<methodology-repo>/docs/OPERATIONS.md`.
