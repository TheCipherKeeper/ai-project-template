# AGENTS.md — интерфейс

Клиентское приложение. Рабочий цикл —
`<methodology-repo>/docs/WORKFLOW.md`, системные границы —
`<methodology-repo>/docs/ARCHITECTURE.md`.

## Правила

- Выполняй только текущую задачу из `<hub>/BACKLOG.md`.
- Работай через `feat/TASK-NNNN-<slug>` и PR; прямой коммит в `main` запрещён.
- Интерфейс обращается только к зарегистрированному клиентскому сервису-шлюзу.
- Обновляй UI, тесты и `docs/ARCHITECTURE.md` вместе.
- Не храни секреты в клиентской сборке и репозитории.
- Доступность, состояния загрузки, отсутствия данных и ошибки, а также проверка прав входят в независимую проверку
  применимых изменений.
- `.pipeline.json` содержит точные команды и выходы исполнимого конвейера; `deploy`
  завершается проверкой готовности и быстрой проверкой.
- `.env`, файлы блокировки зависимостей и зависимости не меняй без разрешения.
- Временная заготовка или TODO не считается готовым результатом.
- Все диаграммы оформляются только блоками `mermaid`; текстовые псевдосхемы и
  другие языки диаграмм запрещены.

## Проверка

```bash
pnpm lint && tsc --noEmit
pnpm test
pnpm build
```

Дополнительно запускаются средство проверки методологии и проверки по условиям из
`<methodology-repo>/docs/REFERENCE.md`. Интеграция — слияние со свёрткой коммитов; развёртывание в тестовой среде
идентифицируется SHA коммита.
