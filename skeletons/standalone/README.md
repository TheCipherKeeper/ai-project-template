# <standalone-component>

Репозиторий независимо поставляемой программы вне сервисного обмена. Её
назначение, форма поставки, параметризация и наблюдаемые поверхности канонически
описываются в `docs/ARCHITECTURE.md`. Это рабочий компонент, а не временная
заглушка.

## Первичная настройка

Выполните один раз в указанном порядке:

1. Получите у человека назначение и высокоуровневую границу компонента.
2. Выберите стек и форму поставки: `container`, `cli` или явно заданную другую.
3. В `.methodology.yml` закрепите точный тег или коммит методологии.
4. Заполните `docs/ARCHITECTURE.md`, включая параметры и поверхности, если они
   действительно существуют.
5. Зафиксируйте в `AGENTS.md` точные команды форматирования, статического анализа,
   проверки типов, тестирования и сборки.
6. Для формы `container` настройте `.env.example`, `Dockerfile` и
   `docker-compose.yml`; для другой формы удалите неприменимые контейнерные файлы.
7. Задайте переменные CI `METHODOLOGY_REPOSITORY` и `METHODOLOGY_VERSION`.
8. Укажите лицензию ниже и выполните локальные проверки.

## Карта файлов

| Файл | Назначение | Режим изменения |
|---|---|---|
| `docs/ARCHITECTURE.md` | единственный реестр границы, формы, параметров и поверхностей | вместе с компонентом |
| `.methodology.yml` | машинный тип `standalone` и закреплённая методология | при создании или обновлении методологии |
| `AGENTS.md` | локальные ограничения и точные команды | при создании или изменении набора инструментов |
| `.env.example`, `Dockerfile`, `docker-compose.yml` | контейнерная форма поставки | только для формы `container` |
| `.github/workflows/verify.yml` | обязательная проверка методологии | вместе с общей заготовкой CI |
| `.evidence/` | записи выполнения | только автоматизация |

Компонент не подключается к брокеру и не выдаётся за сервис. Системный состав и
задачи находятся в хабе.

## Локальный запуск контейнерной формы

```bash
cp .env.example .env
docker compose up --build
```

Для другой формы команда фиксируется в `AGENTS.md`. Рабочий цикл —
`<methodology-repo>/docs/WORKFLOW.md`, эксплуатация —
`<methodology-repo>/docs/OPERATIONS.md`.
