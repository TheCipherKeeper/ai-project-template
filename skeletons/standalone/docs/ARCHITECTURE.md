# Архитектура автономного компонента

> Шаблон одного автономного компонента. Компонент выполняется как отдельная
> программа, не подключается к брокеру и не является сервисом системы. Человек
> задаёт форму поставки: контейнер, программа командной строки или иной вариант
> поставки.
> Способ развёртывания зависит от формы: контейнер использует `Dockerfile`, а CLI
> поставляется как артефакт сборки. Подробности — в
> `<methodology-repo>/docs/OPERATIONS.md`.
> При наличии наблюдаемой поверхности внешний наблюдатель проверяет её по
> отдельному каналу (см. `<methodology-repo>/docs/ARCHITECTURE.md`).
> `MODULE.md`/`SPEC.md` сюда **не применяются** (канон вариантов использования сервиса;
> компонент описывается как отдельная программа). Заполните секции
> под конкретный компонент.

## Развёртывание

- Развёртывание зависит от формы. Для `container`: контейнер со своим `Dockerfile`, локальная
  разработка — `docker-compose.yml` (этот компонент, **без брокера**). Для `cli`:
  артефакт/сборка. Детали — `<methodology-repo>/docs/OPERATIONS.md`.
- Системный файл Docker Compose (все сервисы и контейнерные автономные компоненты) — в хабе,
  не здесь.
- Структурное соответствие методологии проверяется контрольным рубежом
  (см. `<methodology-repo>/docs/WORKFLOW.md`).
