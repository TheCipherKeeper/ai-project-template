# AGENTS.md — хаб

Хаб хранит состав программы, общие соглашения, единственный `BACKLOG.md`,
системный файл Docker Compose и ADR. Код сервисов, интерфейсов и автономных компонентов
находится в их репозиториях.

## Правила

- Человек добавляет и упорядочивает задачи; агенты выполняют первый `[ ] ready`.
- `BACKLOG.md` — единственный источник истины о задачах; после его изменения
  агент генерирует `.tasks` штатным синхронизатором и не редактирует их вручную.
- Человек определяет состав репозиториев, их количество, назначение и
  высокоуровневые границы; агент только отражает явно принятое решение в хабе.
- `[~]`, блокирующий PR и параллельный захват задач не используются.
- `COMPOSITION.md` описывает компоненты и их версии или хеши артефактов.
- `CONVENTIONS.md` содержит только общесистемные контракты.
- ADR создаётся для труднообратимого общесистемного решения, а не обычного
  рефакторинга.
- Системный файл Docker Compose использует собранные образы компонентов.
- Изменения идут через `feat/TASK-NNNN-<slug>` и слияние PR со свёрткой коммитов в `main`.
- После успешной поставки служебный PR с меткой `task-finalization` может изменить
  только `BACKLOG.md`, соответствующую `.tasks/TASK-NNNN.json` и
  `.evidence/TASK-NNNN.json`. Он проходит `policy`; продуктовые стадии имеют
  результат `not_applicable` и не создают evidence о финализации.
- `.pipeline.json` содержит точные команды исполнимого конвейера; его маркеры и пропуск
  обязательной стадии блокируют слияние.
- Секреты, `.env` и файлы блокировки зависимостей не меняются без разрешения.
- Все диаграммы оформляются только блоками `mermaid`; текстовые псевдосхемы и
  другие языки диаграмм запрещены.

`<methodology-repo>/docs/ARCHITECTURE.md`, эксплуатация —
