# AGENTS.md — хаб

Хаб хранит состав программы, общие соглашения, единственный `BACKLOG.md`,
системный compose и ADR. Код сервисов и интерфейсов находится в их репозиториях.

## Правила

- Человек добавляет и упорядочивает задачи; агенты выполняют первый `[ ] ready`.
- Человек определяет состав репозиториев, их количество, назначение и
  высокоуровневые границы; агент только отражает явно принятое решение в хабе.
- `[~]`, lock-PR и параллельный захват задач не используются.
- `COMPOSITION.md` описывает компоненты и их версии/digest.
- `CONVENTIONS.md` содержит только общесистемные контракты.
- ADR создаётся для труднообратимого общесистемного решения, а не обычного
  рефакторинга.
- Системный compose использует собранные образы компонентов.
- Изменения идут через `feat/TASK-NNNN-<slug>` и squash PR в `main`.
- Секреты, `.env` и lock-файлы не меняются без разрешения.

Цикл задачи — `<methodology-repo>/docs/WORKFLOW.md`, архитектурный канон —
`<methodology-repo>/docs/ARCHITECTURE.md`, эксплуатация —
`<methodology-repo>/docs/OPERATIONS.md`.
