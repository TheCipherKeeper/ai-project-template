# Свидетельства выполнения

После успешной поставки автоматизация добавляет `.evidence/TASK-NNNN.json` по схеме
`<methodology-repo>/schemas/evidence.schema.json` служебным PR `task-finalization`.
Средство проверки связывает его с машинной задачей и закреплённой версией методологии.
Свидетельство версии 2 содержит массив `deliveries` с одной поставкой на каждый
целевой репозиторий задачи. Поля `repository` и `commit` каждой поставки указывают
продуктовый репозиторий и его merge SHA, а не коммит хаба. Свидетельства версии 1
читаются только для обратной совместимости; новые свидетельства создаются в версии 2.

Поле `source` указывает на журнал проверки, `reviewer` — на независимый
вызов проверяющего, `artifact` поставки содержит хеш, а `retained_until` задаёт срок, до
которого CI обязан сохранять свидетельство и связанные журналы. Секреты и персональные
данные сюда не записываются; исполняемые артефакты хранятся в CI, а не в Git.
Финализация не создаёт свидетельство о самой себе и не запускает поставку.
