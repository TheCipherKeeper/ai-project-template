# Список задач

Человек задаёт порядок и продуктовый замысел. Агенты выполняют только первый
пункт `[ ] ready` и не начинают следующий до завершения или диагностической
остановки текущего. Рабочий PR показывает, что задача выполняется; `[~]` и
блокирующий PR не используется.

Правила цикла: `<methodology-repo>/docs/WORKFLOW.md`.
`BACKLOG.md` — единственный источник истины о задачах; машинная проекция задач
не используется, `.tasks/` не ведётся.
Человек создаёт минимальную задачу. Агент добавляет риск, автономность, триггеры
и откат отдельным служебным PR `task-qualification` до начала реализации.

## Задачи

### TASK-0001. [ ] ready — Краткое название

Целевые репозитории:
- https://github.com/owner/repository

Цель:
Опиши наблюдаемый результат для пользователя или системы, не способ реализации.

Готово, когда:
- результат можно проверить;
- важный граничный случай учтён.

Не входит:
- явное ограничение или «нет».

## Выполнено

После успешной поставки агент создаёт служебный PR `task-finalization`, в котором
одновременно помечает задачу `[x]` и добавляет `.evidence/TASK-NNNN.json`. Задача
считается завершённой после успешного `policy` и слияния этого PR в `main` хаба.
