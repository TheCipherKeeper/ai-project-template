# Архитектурные профили — справочник

Универсальные agent-safety правила (DoR, risk, роли, evidence, fail-closed,
ветвление и recovery) отделены от архитектурных решений конкретной программы.

Текущий обязательный профиль называется `broker-microservices-v1`:

- отдельные репозитории хаба, сервисов и интерфейсов;
- service-to-service только через брокер;
- один gateway для browser-facing HTTP/WS;
- один язык на сервис;
- usecase/ports/domain/adapters внутри сервисного модуля.

Это осознанный профиль этой методологии, а не универсальное свойство агентной
разработки. Смена профиля требует отдельной версии методологии, миграционного
гайда и обновления всех затронутых skeletons. Локальное отклонение программы —
ADR; тихое смешение профилей запрещено.
