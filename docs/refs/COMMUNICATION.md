# Общение микросервисов (референс)

Сервисы общаются **только через брокер** (один на систему: Kafka / Redpanda /
NATS). Никаких прямых service-to-service вызовов в обход брокера. Формат
сообщений (event envelope) задаёт **хаб** в `CONVENTIONS.md`, сервисы его
потребляют на пиннённой версии.

> Референс (факт «как сервисы общаются»). Где описать топики конкретного
> сервиса — `docs/guide/10-architecture.md` → *Брокер*. Полная модель
> верификации контрактов — `docs/refs/VERIFICATION.md`. Скелет `CONVENTIONS.md`
> для хаба — `skeletons/hub/CONVENTIONS.md`.

## Брокер

- **Один брокер на систему** (Kafka / Redpanda / NATS — зафиксируй в хабе).
- Сервис — **клиент брокера**: публикует и/или читает топики.
- Адрес для локальной разработки — из `docker-compose.yml` сервиса (сервис
  `broker`); для системы — из системного compose хаба.

## Event envelope

- Формат сообщений — из хаба `CONVENTIONS.md` (или `docs/CONVENTIONS.md` в
  standalone-сервисе без хаба). **Не изобретать свой формат в спеках сервисов.**
- Хаб экспонирует контракты с версиями: `CONVENTIONS@v1`, `@v2`, …
- Сервис **пинит** версию в своей конфигурации/манифесте
  (`docs/ARCHITECTURE.md` → *Контракты хаба*); гейт проверяет сервис против
  **пиннённой** версии, не HEAD.
- Изменение хаба в `@v2` не красит сервисы на `@v1`. Миграция — отдельным PR
  сервиса: бамп пина `v1→v2` + правки под новый контракт.

## Топики

Топики конкретного сервиса (publish/consume) — в `docs/ARCHITECTURE.md` →
*Брокер* (таблица: топик / направление / назначение). Спеки модулей ссылаются
на эти топики, не дублируют их.

## Границы

- Прямая связность сервисов в обход брокера (HTTP/RPC/общая БД между сервисами)
  — **запрещена**. Это выход за границу одного сервиса; решается в хабе/ADR.
- Доверительная граница сервиса (где проходит, гарантии, подпись/аутентификация)
  — в `docs/ARCHITECTURE.md` → *Доверительная граница*.

## Связь с верификацией

Ребро `хаб → сервис` проверяет соответствие хабу на пиннённой версии контрактов;
без пина ребро «вниз» нерабочее. Conformance (топики/envelope) — агент,
неблокирует; behavioral (собирается/тестится ли против контракта) —
детерминированный CI, блокирует. Полная модель — `docs/refs/VERIFICATION.md`.
