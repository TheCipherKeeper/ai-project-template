# Миграции данных — справочник

Изменения данных выполняются по expand → migrate → contract. Разрушительная
смена схемы и зависимый от неё код не выпускаются одним шагом.

План обязан содержать объём и время, блокировки, backup/restore, dry run,
совместимость старой и новой версии, проверку результата и rollback либо
roll-forward. Для `high` человек разрешает продвижение после test-deploy; для
`critical` — также merge. Необратимость явно отмечается и не маскируется TODO.
