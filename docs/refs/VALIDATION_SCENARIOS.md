# Acceptance-сценарии методологии — справочник

Перед стабильным релизом методологии проверяется не только целостность файлов,
но и прохождение эталонных state-machine сценариев:

1. Готовая feature-задача проходит до evidence и `[x]`.
2. Неоднозначная задача получает `needs-spec` до захвата.
3. Security finding удерживает PR и не пропускается при недоступности роли.
4. False positive отклоняется только после adversarial-verify.
5. Исчерпание retry даёт evidence и освобождает лок recovery-PR.
6. Падение координатора продолжает тот же `run_id` идемпотентно.
7. Несовместимая event schema блокирует merge.
8. Миграция проходит expand/migrate/contract и risk-gate.
9. Ошибка test-deploy вызывает проверенный rollback и hold.
10. Неполный evidence запрещает `[~]→[x]`.

Сценарий считается автоматизированным, когда имеет fixture, ожидаемые переходы
и CI-тест. До появления fixture он явно отмечается `manual`; выдавать его за
исполняемый rule запрещено.
