# Риск и предел автономности — справочник

Риск определяет не качество реализации, а максимально допустимое действие
автономного pipeline. Агент может повысить риск и понизить автономность, но не
наоборот. Остаточный риск принимает только человек.

| Класс | Примеры | Максимальная автономность |
|---|---|---|
| `low` | доки, локальный рефакторинг без смены поведения | auto-merge + test-deploy |
| `medium` | новое обратимо совместимое поведение без изменения данных | auto-merge + test-deploy |
| `high` | auth, права, публичный контракт, миграция, PII | test-deploy; человек разрешает продвижение дальше |
| `critical` | удаление данных, платежи, production-секреты, необратимое действие | человек до merge и перед deploy |

Уровни автономности: `pr-only`, `human-before-merge`, `auto-test-deploy`,
`human-before-production`. В задаче указывается наиболее строгий применимый
уровень. Изменение уровня отдельным агентом запрещено; повышение полномочий —
новое решение человека с audit trail.

Обязательная роль или rule недоступны — **fail closed**. Повтор ограничен
политикой `RETRY.md`; затем задача получает `automation-failed`. Fail-open
разрешён только явно указанной необязательной проверке low-risk задачи и не
применим к security, контрактам, миграциям и данным.
