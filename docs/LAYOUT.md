# Раскладка каталогов

Один микросервис в одном репозитории, один стек (см. `docs/STACKS.md`).
Дерево ниже — общая форма; конкретные имена каталогов исходников зависят от
выбранного стека (Python: `src/`, Go: `cmd/`+`internal/`, Rust: `crates/` или
`src/`, TS: `src/`).

```
<service>/
  AGENTS.md                 правила работы
  README.md                 описание сервиса, быстрый старт
  docs/
    INDEX.md                карта документации
    ARCHITECTURE.md         архитектура сервиса
    BACKLOG.md              очередь задач
    STACKS.md               toolchain/layout выбранного стека
    LAYOUT.md               этот файл
    specs/                  контракты модулей сервиса (по одному на модуль)
    adr/                    архитектурные решения
  <src-раздел>              исходники — по конвенции выбранного стека
  <tests-раздел>            тесты — по конвенции стека
  <manifest>                манифест стека (pyproject.toml / go.mod /
                            Cargo.toml / package.json)
  <lock>                    lock-файл (коммитится)
  .gitignore
  LICENSE  LICENSE-DOCS
```

## Модули и спеки

Внутри сервиса код делится на модули. **Каждый модуль — отдельный спек** в
`docs/specs/<модуль>.md` по канонической структуре (см. `docs/INDEX.md` →
*Как работать со спеками*). Модуль — это:

- Python: пакет/подпакет под `src/<service>/`.
- Go: пакет под `internal/` или `pkg/`.
- Rust: crate (если workspace) или модуль под `src/`.
- TypeScript: директория/модуль под `src/`.

## Где живут общие вещи

- Контракты модулей — `docs/specs/`.
- Архитектура (потоки, границы) — `docs/ARCHITECTURE.md`.
- ADR — `docs/adr/` (или внешний хаб — см. `AGENTS.md` → *ADR*).
- Кросс-репо конвенции (event envelope, форматы) — во внешнем хабе, если есть;
  иначе заведи `docs/CONVENTIONS.md` и сошлись на него из спеков.

## Несколько сервисов

Если система состоит из нескольких микросервисов — **каждый в своём репо** со
своим экземпляром этого шаблона. Общее (состав, ADR, конвенции) — в хабе.
Этот шаблон описывает один сервис, не монорепо сервисов.
