# Фаза 10 — Описать архитектуру сервиса

**Цель:** заполнить `docs/ARCHITECTURE.md` — роль сервиса, модули, брокер,
топики, потоки данных, доверительная граница.

**Когда:** после бутстрапа (фаза 00), до написания кода и спек модулей.

**Что нужно до:** фаза 00 (выбран стек, заведён workspace).

## Шаги

Пройди по секциям `docs/ARCHITECTURE.md` (скелет-комментарии подсказывают, что
писать в каждой):

1. **Что это** — 1–2 предложения: какой сервис, какую роль в программе играет.
   Ссылка на хаб `COMPOSITION.md` для системного контекста.
2. **Что делает** — нумерованный список ключевых функций (глагол → что/зачем).
3. **Чего не делает** — явные границы. Важно для агентов, чтобы не «помогали»
   там, где не надо.
4. **Модули** — таблица модулей workspace'а (модуль / роль / publish·consume
   топики) + DAG зависимостей между модулями (mermaid).
5. **Брокер** — выбрать один на систему (Kafka / Redpanda / NATS), зафиксировать.
   Адрес (локальная разработка) — из `docker-compose.yml`, сервис `broker`.
   - **Контракты хаба:** `CONVENTIONS@v<N>` — пин версии, по которой гейт
     проверяет сервис (см. `docs/refs/VERIFICATION.md`). Бамп пина — отдельным PR.
   - Топики сервиса: таблица (топик / направление publish|consume / назначение).
   - Формат сообщений (event envelope) — из хаба `CONVENTIONS.md` (или
     `docs/CONVENTIONS.md` в standalone), **не свой**.
6. **Потоки данных** — потоки этого сервиса через брокер (входящие/исходящие),
   mermaid-граф + шаги каждого потока.
7. **Доверительная граница** — где проходит граница доверия сервиса, что по
   какую сторону, гарантии, подпись/аутентификация. Для сервисов без границ —
   секцию убрать.
8. **Деплой / Ссылки** — ссылка на `docs/guide/50-deploy.md` и
   `docs/refs/DEPLOYMENT.md`; ссылки на хаб `COMPOSITION.md`, `CONVENTIONS.md`,
   `adr/`.

## Канон (где правда)

- `docs/ARCHITECTURE.md` — сам файл (скелет-комментарии).
- Хаб `COMPOSITION.md` (состав программы), `CONVENTIONS.md` (event envelope).
- `docs/refs/VERIFICATION.md` — почему пин `CONVENTIONS@v<N>` обязателен.
