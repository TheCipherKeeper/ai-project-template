# Этап 40 — Выполнить проверку перед коммитом

**Цель:** убедиться, что изменение соответствует канону, до коммита.

**Когда:** перед каждым коммитом; всегда после изменения кода. Гейт в CI
триггерится на каждый коммит (включая merge и прямой push) — дрейф в обход
гейта невозможен.

**Что нужно до:** локально ты на ветке `feat/<задача>`, есть изменения. `main`
проверяется CI после merge; прямой локальный коммит в `main` запрещён.

**Применимо:** всем типам инстанцированных репозиториев. Конкретный набор
инвариантов выбирается по матрице `docs/refs/VERIFICATION.md` → *Применимость
инвариантов по типу репо*.

**Результат:** локальные воспроизводимые проверки пройдены; применимые
инварианты проверены; подтверждённые agent-находки разрешены или PR поставлен
на hold с задачей в бэклоге.

## Шаги

### 1. Воспроизводимые проверки

Три группы, **все** должны пройти: **lint** (формат + линтер), **test**,
**build**. Команды твоего выбранного стека — твоя закреплённая строка в
`AGENTS.md` → *Команды проверки (выбранный стек)* (полная конфигурация
toolchain'а — `docs/refs/STACKS.md`).

Любая fail → чини, повтори. **Не коммить.**

### 2. Перечень инвариантов

Пробеги по чеклисту. `rule` — детерминированная проверка (скрипт/CI),
блокирует. `agent` — семантическая (LLM): **сырой вердикт не блокирует**;
survived-находка разрешается (авто-фикс / `BACKLOG`+холд), «в комментарий в
воздух» не уходит — в автономном цикле его некому читать. Полный чеклист с
пояснениями — `docs/refs/VERIFICATION.md`; здесь — компактная форма для
прогона:

**rule (блокируют):**
1. Локально ветка `feat/<task>`; `main` допустим только как CI-контекст после merge.
2. Conventional Commits + scope.
3. `lint / test / build` проходят (включая adversarial-тесты тестировщика).
4. Дифф тронул спеку «TODO→есть» → тронул код/тест модуля.
5. Программный бэклог хаба (`<hub>/BACKLOG.md`): `[x]` на выполненных; ровно
   один `[~]` in-flight на всю программу (глобальный лок); без перепрыгивания;
   punt (needs-human) съезжает с верхушки — единственное исключение.
11. ADR-ссылки из ARCHITECTURE существуют в хабе.
12. Модуль экспонирует каноничные швы (`usecases/`+`ports`+`domain`+`adapters/`
    по per-stack имён); каждый юзкейс определяет input port.
15. vuln-scan зависимостей/секретов проходит (rule-часть #15).

**agent (сырой вердикт не блокирует; подтверждённая находка требует разрешения):**
6. Каждый пункт «Что есть» в спеке → есть тест (тестировщик, независимый вызов).
7. Содержимое «Что есть» соответствует коду.
8. ARCHITECTURE (роль/топики/граница) соответствует хабу (`CONVENTIONS`).
9. Stub не выдаётся за реализацию (placeholder/TODO помечены).
10. Доки не противоречат: внутри яруса (в т.ч. `guide/` против `refs/`) —
    дефект (чинят к одной правде), между ярусами — старший побеждает.
13. Направление зависимостей: usecases → только ports+domain, не adapters;
    adapters реализуют ports; модуль реэкспортит только input ports.
14. Интерфейс зарегистрирован в `COMPOSITION` хаба; заявленные им
    consumed-эндпоинты (из `ARCHITECTURE` интерфейса) существуют в
    `ARCHITECTURE` **gateway-сервиса** (единственного browser-facing surface).
15. threat-model: нет очевидных auth/secret-injection/input-trust/доверие-граница
    дыр (agent-часть #15, безопасник).

### 3. Правила чтения правил

- Гейт читает правила из **авторитета** (методология/хаб), не из копии в репо —
  иначе рассинхронизация копии с источником не ловится. Поэтому `docs/guide/`
  и `docs/refs/` читаются из репо методологии, а не копируются в сервис.
- Сервис соответствует **текущему контракту хаба** (`CONVENTIONS`); гейт
  проверяет сервис против него.
- Agent-ошибка/недоступность **не** блокирует; путь апелляции обязателен;
  survived-находка разрешается (авто-фикс/`BACKLOG`), «в воздух» не уходит.

### 4. Решение

- rule fail → чини, не коммить.
- conformance-находка (agent) → **разрешается**, не «в комментарий»:
  adversarial-verify (попробуй опровергнуть) →
  · не выжила: выкинуть (false positive);
  · выжила, структурная: авто-фикс новым коммитом → re-гейт;
  · выжила, неструктурная: в `BACKLOG`, PR на холд.
- Bounded retries: N попыток фикс→recheck; дальше — в `BACKLOG`, не
  зацикливаться.
- Каденс: rule — на push (дёшево); agent — на PR, не на каждый коммит.
- Ревьюер — независимый вызов агента (не имплементер); человек в pre-deploy
  не участвует (`docs/refs/PIPELINE.md`).

## Канон (где правда)
