# Фаза 30 — Взять задачу и реализовать (рабочий цикл)

**Цель:** выполнить один пункт бэклога — от ветки до PR.

**Когда:** есть невыполненный пункт в `docs/BACKLOG.md`, нужно писать код.

**Что нужно до:** фаза 10 (архитектура) и фаза 20 (спека нужного модуля).

> **Бэклог — строгая последовательная очередь** (как миграции). Запрещено
> выполнять задачи параллельно или перепрыгивать через невыполненный пункт.
> Задача N+1 начинается только после полного завершения задачи N.

## Шаги

1. **Взять задачу.** Прочитай `docs/BACKLOG.md`, возьми **самый первый**
   невыполненный `[ ]` пункт. Тег задачи указывает модуль
   (`docs/specs/<module>.md`).
2. **Прочитать контекст.** `docs/ARCHITECTURE.md` → спеку нужного модуля
   (возможно нескольких).
3. **Ветка от main:**
   ```bash
   git checkout main && git pull && git checkout -b feat/<задача>
   ```
   Модель ветвления — `AGENTS.md` → *Модель ветвления* (коротко: `feat/<задача>`
   от `main`, удаляется после merge; прямой коммит в `main` запрещён; релизы —
   тегами `vX.Y.Z`, `docs/guide/70-release.md`).
4. **Код + тесты.** Реализуй по пунктам «Что есть» в спеке; напиши тесты
   на каждый пункт «Что есть».
5. **Проверить перед коммитом** — фаза `docs/guide/40-verify.md`
   (lint / test / build + invariant-чеклист). Все блокирующие должны пройти;
   иначе — чини, повтори. **Не коммить с упавшим гейтом.**
6. **Обновить спеку:** перенеси реализованное из «Что TODO» в «Что есть».
7. **Обновить BACKLOG:** поставь `[x]` на выполненный пункт (на первом
   невыполненном — без перепрыгивания).
8. **Коммит** — Conventional Commits, scope = имя модуля (или `deploy`/`docs`):
   ```
   feat(<module>): add zfs-snapshot fs probe
   fix(<module>): reject path traversal in HostBridge
   docs: update ARCHITECTURE.md with module matrix
   refactor(<module>): extract envelope signing
   ```
   Breaking changes — `BREAKING CHANGE:` в теле. Язык коммитов — `AGENTS.md` →
   *Язык*.
9. **Пуш, открыть PR в `main`.** Ревьюер — независимый агент (не ты); merge —
   по зелёному гейту, без человеческого одобрения. Conformance-находки
   разрешаются авто-фиксом или уходят в `BACKLOG` (фаза 40,
   `docs/refs/PIPELINE.md`). После merge — ветка удаляется.

## Канон (где правда)

- `AGENTS.md` — правила коммитов/ветвления, что можно/нельзя, язык.
- `docs/guide/40-verify.md` — что именно проверять перед коммитом.
- `docs/refs/VERIFICATION.md` — полная теория гейта (если нужно).

## Что после

- PR в `main` → агент-ревью (фаза 40) → merge по гейту → ветка удаляется.
- Auto-rc → деплой в тестовую среду; баг из теста → `BACKLOG` → следующий
  пункт (`docs/refs/PIPELINE.md`).
- Следующий пункт BACKLOG (фаза 30 повторяется; очередь строго сверху).
