# Этап 05 — Подготовить задачу к автономной работе

**Цель:** превратить продуктовый intent в однозначную проверяемую задачу.

**Применимо:** всем задачам инстанцированных репозиториев до захвата лока.

1. Заполнить формат из `docs/refs/TASK.md` в `<hub>/BACKLOG.md`.
2. Назначить риск и предел автономности по `docs/refs/RISK.md`.
3. Указать контракты, данные, rollback и acceptance criteria.
4. Запустить независимую Definition of Ready-проверку.
5. Если есть неоднозначность — назначить точный диагностический статус и задать
   человеку минимальный вопрос. Иначе оставить `[ ] ready`.

**Результат:** первый доступный пункт имеет статус `ready` и может быть атомарно
захвачен этапом 30.
