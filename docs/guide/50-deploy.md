# Этап 50 — Запустить сервис локально

**Цель:** поднять локальный `docker-compose` (брокер + этот сервис) и
убедиться, что сервис работает с брокером.

**Когда:** нужно прогнать интеграционно; перед деплоем; после изменений в
`Dockerfile` / `docker-compose.yml` / брокер-интеграции.

**Что нужно до:** этап 00 (compose настроен под выбранный стек и брокер).

**Применимо:** сервис и gateway-сервис. Для интерфейса и автономного компонента
используй соответствующие разделы `docs/refs/DEPLOYMENT.md`.

**Результат:** локальный compose поднимает брокер и один сервис, конфигурация
валидна, секреты не попали в репозиторий.

## Шаги

```bash
cp .env.example .env          # заполни переменные (брокер-адрес, per-сервис конфиг)
docker compose up --build      # брокер + этот сервис (собирает Dockerfile из корня)
docker compose config          # проверить валидность compose (опц., перед up)
docker compose down            # остановить
```

1. **`.env`** — скопируй `.env.example`, заполни. Имена ключей — единственный
   источник в `.env.example`. Секрета в репо не держать.
2. **`docker compose up --build`** — поднимает `broker` + этот сервис
   (контейнер из `Dockerfile` в корне). Сервис зависит от брокера
   (`depends_on: [broker]`).
3. **Healthcheck** — брокер: команда по выбранному в хабе брокеру (Redpanda /
   Kafka / NATS — таблица в `docs/refs/DEPLOYMENT.md` → *Healthcheck брокера*).
   Сервис — если предоставляет healthcheck.
4. **Внешние порты** открывай осознанно — только если сервис принимает трафик
   снаружи.
5. **Остановка** — `docker compose down`.

> Системный compose (все сервисы программы вместе) — в **хабе**, не здесь. Он
> ссылается на собранные образы сервисов. Локальный compose в репо сервиса —
> только локальная разработка одного сервиса.

## Канон (где правда)

- `docs/refs/DEPLOYMENT.md` — структура compose, требования к Dockerfile, env.
- `docker-compose.yml` (корень) — сам compose.
- `docs/ARCHITECTURE.md` → *Брокер* — топики, адрес.

## Что после

- Собранный образ подхватывается системным compose в хабе (вне репо сервиса).
- Перед деплоем в хаб: прогнать `lint / test / build` + `docker compose build`
  (этап 40 + `docs/refs/DEPLOYMENT.md`).

Критерий завершения: `docker compose config` проходит, сервис стартует и
работает с брокером, остановка не требует ручной очистки состояния.
