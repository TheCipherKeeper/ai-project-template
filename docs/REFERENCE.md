# Краткий справочник

## Статусы задачи

| Статус | Значение |
|---|---|
| `[ ] ready` | первая такая задача доступна агентам |
| `[x]` | служебный PR финализации слит в `main` хаба: поставка успешно проверена, а задача и каноническое свидетельство согласованы |
| `needs-input` | требуется продуктовое решение человека |
| `blocked-external` | недоступна внешняя система или доступ |
| `automation-failed` | обязательный инструмент или CI не работает |
| `retry-exhausted` | исчерпан ограниченный цикл исправлений |

Статуса `[~]` и распределённой блокировки нет.

Канонический заголовок открытой задачи — `### TASK-NNNN. [ ] <status> — Название`,
где `<status>` — `ready` или один из диагностических статусов. Завершённая задача
использует `### TASK-NNNN. [x] — Название`. Диагностика записывается сразу под
заголовком в строке `Диагностика: краткая причина`; поля цели, критериев и объёма
изменений сохраняются. Синхронизация переносит причину в машинное поле `diagnostic`.

## Риск и автономность

| Риск | Примеры | Максимальное действие агента |
|---|---|---|
| `low` | документация, локальный рефакторинг | слияние и развёртывание в тестовой среде |
| `medium` | обратимо совместимая функция без изменения данных | слияние и развёртывание в тестовой среде |
| `high` | аутентификация, права, персональные данные, публичный контракт, миграция | развёртывание в тестовой среде; человек разрешает эксплуатационную среду |
| `critical` | удаление данных, платежи, секреты эксплуатационной среды, необратимое действие | слияние и развёртывание в тестовой среде; человек разрешает эксплуатационную среду |

Агент может повысить риск и понизить автономность, но не наоборот. Он не может
ослабить обязательную проверку, принять остаточный риск или расширить собственные полномочия.
Независимая проверка обязательна для всех уровней риска. Неопределённость
трактуется в сторону более строгого класса. Недоступная обязательная проверка
блокирует работу.

## Триггеры проверок

| Изменение | Дополнительная проверка |
|---|---|
| Любой продуктовый PR | проверка правил + независимая проверка |
| Документация, локальный рефакторинг | без дополнительной проверки |
| Обычная функциональность | без дополнительной проверки |
| Сложные состояния или интеграция | сценарная проверка |
| API, схема события | проверка совместимости + контрактные тесты |
| Аутентификация, права, секреты, сеть, файлы, ввод | проверка безопасности + сканирование |
| Сохраняемые данные | пробный запуск миграции + откат или исправление вперёд |
| Заданный человеком новый репозиторий или системная граница | проверка архитектуры; возможно ADR |

Триггер повышает минимальный риск: безопасность, публичный контракт и сохраняемые данные
не могут быть ниже `high`; необратимое действие не может быть ниже `critical`.

## Команды стеков

| Стек | Проверка | Тест | Сборка |
|---|---|---|---|
| Python | `ruff format --check . && ruff check . && pyright` | `pytest` | `uv build` |
| Go | `gofmt -l . && go vet ./...` | `go test ./...` | `go build ./...` |
| Rust | `cargo fmt --check && cargo clippy -- -D warnings` | `cargo test` | `cargo build --release` |
| TypeScript | `pnpm lint && tsc --noEmit` | `pnpm test` | `pnpm build` |

Репозиторий выбирает один стек и фиксирует точные команды в своём `AGENTS.md`.
Сервис использует только Python, Go или Rust; TypeScript допустим для интерфейса
или автономного компонента.

## Git

Точный Git-цикл задачи и условия остановки заданы только в
[`WORKFLOW.md`](WORKFLOW.md). Формат стабильного выпуска задан только в
[`OPERATIONS.md`](OPERATIONS.md). Этот справочник не вводит дополнительных правил
перехода.

## Задача

```markdown
### TASK-NNNN. [ ] ready — Краткое название

Целевой репозиторий: имя репозитория
Риск: low
Автономность: auto-test-deploy
Триггеры:
- нет

Цель:
Наблюдаемый результат для пользователя или системы.

Готово, когда:
- проверяемый критерий;
- проверяемый критерий.

Не входит:
- явное ограничение или «нет».

Откат: вернуть предыдущий хеш артефакта.
```

Человек задаёт цель, критерии готовности, ограничения и границы репозиториев.
Агент определяет риск, автономность, триггеры, целевые артефакты, технический план,
тесты и способ отката; человек может сделать классификацию строже.
`BACKLOG.md` является единственным источником истины о задачах; машинная проекция
не используется, `.tasks/` не ведётся.

## Язык терминов

Поясняющий текст документации пишется по-русски. Без перевода сохраняются только
точные идентификаторы и значения (`repository_type`, `standalone`), команды,
пути, имена технологий и официальные названия стандартов. Если английское имя
нужно для сопоставления с кодом, сначала указывается русское понятие, затем точное
имя в обратных кавычках.
