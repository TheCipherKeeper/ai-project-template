# <модуль>

> Пример канонической структуры спека. Скопируй под свой модуль, удали этот
> блок-комментарий. Спек описывает контракт (что), не реализацию (как).
> Канон структуры (7 секций) и чек-лист — `docs/refs/SPEC.md`; процедура
> заведения модуля — `docs/guide/20-define-module.md`. Модуль = каталог
> выбранного стека (см. `docs/refs/LAYOUT.md`); команды проверки —
> в `AGENTS.md` → *Команды проверки (выбранный стек)*.

Краткое описание модуля: какую роль играет, какие границы ответственности.

## Интерфейсы

- `Module::new(args)` — создание.
- `Module::do_thing(input) -> Result<Output, ModuleError>` — основная операция.
