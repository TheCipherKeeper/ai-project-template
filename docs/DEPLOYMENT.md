# Деплой

Сервис деплоится **контейнером** со своим `Dockerfile` в корне репо.
`docker-compose.yml` в корне — **только для локальной разработки**: поднимает
брокер и этот сервис. Системный compose (все сервисы программы вместе)
живёт в **хабе** (или отдельном деплой-репо) и ссылается на собранные образы
сервисов — не здесь.

## Запуск локально (брокер + этот сервис)

```bash
cp .env.example .env          # заполни переменные
docker compose up --build     # брокер + сервис
docker compose down           # остановить
```

## Структура docker-compose.yml (локальный dev)

```yaml
services:
  broker:              # один на систему (Kafka / Redpanda / NATS)
    image: <broker-image>
    ports: ["9092:9092"]
    volumes: ["broker-data:/data"]

  <service>:            # этот сервис — контейнер из Dockerfile в корне
    build: { context: . }
    depends_on: [broker]
    env_file: .env
    # внешние порты — только если сервис принимает трафик снаружи

volumes:
  broker-data:
```

Правила:

- `build.context: .` — Dockerfile в корне репо.
- `depends_on: [broker]` — сервис читает/пишет в брокер.
- Переменные — из `.env` / `.env.example`; секрета в репо не держать.
- Внешние порты открывать осознанно.
- Это **локальный** compose; production/системный compose — в хабе.

## Dockerfile сервиса

Собирается из команд `build` выбранного стека (`docs/STACKS.md`). Общие
требования:

- Мультистадийная сборка: stage сборки → тонкий runtime-образ.
- В runtime-образе — только артефакт `build` и runtime-зависимости.
- Не запускать от root, где позволяет стек.
- Healthcheck, если сервис его предоставляет.

## Переменные окружения

`/.env.example` — шаблон, коммитится. `/.env` — реальные значения, в
`.gitignore`. Брокер-адрес и per-сервис конфигурация.

## Сборка и проверка перед деплоем

- Прогнать `lint / test / build` (команды — `AGENTS.md` → *Команды проверки по стекам*).
- `docker compose build` — убедиться, что образ собирается.
- `docker compose config` — проверить валидность compose.
- Собранный образ далее подхватывается системным compose в хабе.
