# Указатель документов

Начальная точка работы с методологией. Выберите рабочую ситуацию и откройте
указанный документ. Каждая процедура из `guide/` самодостаточна. Канонические
технические сведения находятся в `refs/`.

> Центральная методология (читается как гайд) + `skeletons/` (стартовые наборы
> для копирования). Хаб/сервисы — отдельные репо из `skeletons/`; этот репо —
> корень авторитета.

## Выбери тип репозитория

| Я работаю с… | Начальная точка | Процедуры |
|---|---|---|
| Методологией | корневой `AGENTS.md` | проверка документов из `AGENTS.md`; `guide/` к самой методологии не применяется |
| Хабом | `skeletons/hub/AGENTS.md` | общие этапы 30, 40, 60, 70; рабочие артефакты хаба |
| Сервисом или gateway-сервисом | `skeletons/service/AGENTS.md` | полный цикл 00–70 |
| Пользовательским интерфейсом | `skeletons/interface/AGENTS.md` | собственная инициализация из скелета + общие этапы 30, 40, 60, 70 |
| Автономным компонентом | `skeletons/stub/AGENTS.md` | собственная инициализация из скелета + общие этапы 30, 40, 60, 70 |

Матрица обязательных артефактов и требований для всех типов —
`docs/refs/TOPOLOGY.md` → *Матрица применимости*. Каждая процедура также явно
указывает свою применимость и результат.

## Ситуация → читай

Этапы 00, 10, 20 и 50 относятся к сервису (включая gateway); этапы 30, 40, 60
и 70 — общий жизненный цикл инстанцированных репозиториев с оговорённой в
самом документе применимостью.

| Мне нужно… | Читай |
|---|---|
| Войти в проект, поднять сервис с нуля | `docs/guide/00-bootstrap.md` |
| Описать архитектуру (роль, модули, брокер, топики, граница) | `docs/guide/10-architecture.md` |
| Добавить модуль / написать спеку | `docs/guide/20-define-module.md` |
| Спроектировать модуль изнутри (usecases/ports/domain/adapters) | `docs/guide/20-define-module.md` + `docs/refs/MODULE.md` |
| Взять задачу из бэклога и реализовать (рабочий цикл) | `docs/guide/30-implement-task.md` |
| Выполнить проверку перед коммитом | `docs/guide/40-verify.md` |
| Запустить локально (брокер + сервис) | `docs/guide/50-deploy.md` |
| Зафиксировать архитектурное решение (ADR) | `docs/guide/60-adr.md` |
| Выпустить кандидата или стабильную версию | `docs/guide/70-release.md` |
| Создать пользовательский интерфейс и описать его обращения к API | `skeletons/interface/` + `docs/refs/COMMUNICATION.md` → *Сервис-шлюз* / *Пользовательский интерфейс* |
| Назначить сервис-шлюз, предоставляющий клиентский API | создать сервис из `skeletons/service/`; роль зафиксировать в `COMPOSITION` хаба; см. `docs/refs/COMMUNICATION.md` → *Сервис-шлюз* |
| Создать автономный компонент | `skeletons/stub/` + `docs/refs/COMMUNICATION.md` → *Автономный компонент* |
| Узнать правила работы над методологией (ветвление, можно/нельзя, docs-verify, коммиты, язык) | `AGENTS.md` |

## Инстанциация → skeletons/

| Хочу… | Беру |
|---|---|
| Создать хаб-репо (системные контракты, состав, compose) | `skeletons/hub/` |
| Создать сервис-репо (AGENTS, README, compose, env, Dockerfile, docs/) | `skeletons/service/` |
| Создать репозиторий пользовательского интерфейса | `skeletons/interface/` |
| Назначить gateway-сервис (browser-facing, из сервисов) | `skeletons/service/` (роль gateway — в `COMPOSITION` хаба) |
| Создать репозиторий автономного компонента | `skeletons/stub/` |

Шаблоны копируются в новый репозиторий целиком; хаб, сервисы, интерфейсы и автономные компоненты **не
несут** guide/refs, а ссылаются на этот репо методологии
(`<methodology-repo>/docs/...`).

## Рабочие артефакты

| Артефакт | Где | Что |
|---|---|---|
| `docs/ARCHITECTURE.md` | сервис-репо | Архитектура сервиса: модули, брокер, потоки, граница |
| `docs/specs/<module>.md` | сервис-репо | Контракты модулей — по одному на модуль |
| `BACKLOG.md` | **хаб** | Программный бэклог: единая очередь «что делать» по всей программе, глобальный лок (`[~]`) |
| `COMPOSITION.md`, `CONVENTIONS.md` | хаб | Состав программы и event envelope |
| `<hub>/adr/` | хаб | ADR программы (см. `docs/refs/TOPOLOGY.md`) |

## Принципы

- Этап `guide/N-*.md` читается **один**: заголовок-блок (Цель / Когда /
  Что нужно до / Применимо / Результат), шаги, источники фактов и рабочие
  артефакты, критерий завершения, затем следующий шаг.
- Факты — в `refs/` (одна правда). Фаза ссылается на refs, не дублирует.
- `skeletons/` — стартовые наборы, не авторитет фактов (авторитет — `refs/`).
- Приоритет **по ярусам** и арбитраж (внутри-ярусное противоречие — дефект, не
  «старший побеждает»; методология эволюционирует коммитами без ADR) —
  `AGENTS.md` → *Документация (приоритет)*.
