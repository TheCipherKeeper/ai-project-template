# docs/INDEX — роутер документации

Точка входа. Найди свою ситуацию — открой один файл. Каждая фаза `guide/`
**самодостаточна**: читается одна, без чтения остальных. `refs/` — для
деталей по факту.

> Шаблон **одного микросервиса** (один репо = один сервис). Программа целиком
> — в хабе. Этот репо описывает один сервис.

## Ситуация → читай

| Мне нужно… | Читай |
|---|---|
| Войти в проект, поднять сервис с нуля | `docs/guide/00-bootstrap.md` |
| Описать архитектуру (роль, модули, брокер, топики, граница) | `docs/guide/10-architecture.md` |
| Добавить модуль / написать спеку | `docs/guide/20-define-module.md` |
| Взять задачу из бэклога и реализовать (рабочий цикл) | `docs/guide/30-implement-task.md` |
| Проверить перед коммитом (verification gate) | `docs/guide/40-verify.md` |
| Запустить локально (брокер + сервис) | `docs/guide/50-deploy.md` |
| Зафиксировать архитектурное решение (ADR) | `docs/guide/60-adr.md` |
| Узнать правила (ветвление, можно/нельзя, коммиты, язык) | `AGENTS.md` |

## Рабочие артефакты (заполняются под сервис)

| Артефакт | Что |
|---|---|
| `docs/ARCHITECTURE.md` | Архитектура сервиса: модули, брокер, потоки, граница |
| `docs/BACKLOG.md` | Очередь задач (строгая, сверху вниз) |
| `docs/specs/<module>.md` | Контракты модулей — по одному на модуль |
| `docs/adr/` | ADR (если нет хаба; иначе — в хабе) |

## Принципы

- Фаза `guide/N-*.md` читается **одна**. Внутри — заголовок-блок
  (Цель / Когда / Что нужно до / Шаги / Канон / Что после) + инлайн-минимум
  для выполнения в одиночку.
- Факты — в `refs/` (одна правда, без дублирования). Фаза ссылается на refs
  для деталей, не дублирует.
- Рабочие артефакты (`ARCHITECTURE`/`BACKLOG`/`specs`/`adr`) — содержимое
  конкретного сервиса; их заполняют по фазам 10/20/30/60.
- Приоритет доков: хаб > `AGENTS.md` > `guide/`/`refs/` > рабочие артефакты > код.
  Противоречие → старший побеждает, расхождение → ADR (`docs/guide/60-adr.md`).
