# Agent-first Development Methodology

Лёгкая методология для одного живого разработчика и программных агентов. Человек
ставит одну продуктовую задачу за раз и видит результат в тестовой среде; анализ,
код, тесты, review, merge и test deployment выполняют агенты.

## Начать

1. Прочитать [`docs/START.md`](docs/START.md) — модель целиком за один присест.
2. Формулировать задачи по шаблону из
   [`docs/REFERENCE.md`](docs/REFERENCE.md).
3. Для выполнения задачи следовать [`docs/WORKFLOW.md`](docs/WORKFLOW.md).

[`docs/INDEX.md`](docs/INDEX.md) направляет к трём условным справочникам:
архитектуре, эксплуатации и кратким таблицам.

## Состав

- `docs/` — пять коротких документов методологии;
- `skeletons/` — стартовые наборы хаба, сервиса, интерфейса и автономного
  компонента;
- `tools/verify/` — локальный и CI-гейт;
- `schemas/` — машинные форматы отчётов.

## Основной цикл

```text
BACKLOG → реализация → rule gate → условное независимое review
        → merge → test deploy + smoke → done
```

Глобальных локов и статуса `[~]` нет: одновременно выполняется только одна
задача. RC создаётся по необходимости, а не после каждого merge.

## Проверка методологии

```bash
uv run tools/verify/verify.py --report verification.json
```

Изменения идут через `feat/<задача>` и PR в `main`. Прямой коммит в `main`
запрещён. Код распространяется по MIT, документация — CC BY 4.0.
